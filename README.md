# Smart AC Control (NodeMCU to Ferrum FIS07F2) via MQTT

Проект умного управления кондиционером **Ferrum FIS07F2** на базе микроконтроллера NodeMCU (ESP8266) с интеграцией в систему домашней автоматизации по протоколу MQTT. 

Система умеет как отправлять команды на кондиционер, так и перехватывать сигналы с родного ИК-пульта для синхронизации статуса в реальном времени.

## 🚀 Основные возможности
* **Двусторонний обмен данными**: отправка команд по MQTT + чтение сигналов родного пульта ДУ.
* **Синхронизация состояния**: автоматическое обновление JSON-статуса при любых изменениях.
* **Беспроводное обновление**: поддержка OTA-прошивки (Over-the-Air) без отключения от кондиционера.

## 🛠 Схема подключения (Hardware)

Для безопасного сопряжения логических уровней кондиционера (5V) и NodeMCU (3.3V), а также для защиты от помех, используется диодная развязка линии приема. Земли (`GND`) обоих устройств обязательно объединены.

* **ИК-передатчик (Отправка команд)**: 
  * ИК-светодиод подключен к пину **D2 (GPIO4)**. Излучатель направлен непосредственно на фотоприемник кондиционера.
* **ИК-приемник (Синхронизация с пультом)**:
  * Вывод `OUT` встроенного трехногого ИК-приемника кондиционера подключен к пину **D5 (GPIO14)** через быстродействующий диод **FR107**.
  * **Ориентация диода**: Катод (сторона с полоской) подключен к кондиционеру, Анод (сторона без полоски) — к NodeMCU.
  * На пине `D5` программно активирована внутренняя подтяжка `INPUT_PULLUP`.

## 📊 MQTT Интерфейс

### Топик состояния (Выходной)
* `/home/cabinet/ferrum/ac/status` — публикация полного состояния кондиционера в формате JSON при любом изменении.

**Пример структуры JSON:**
```json
{
  "power": false,
  "mode": "auto",
  "fan": "auto",
  "targetTemp": 25,
  "displayOn": true
}
```

### Топики управления (Входные)
Для отправки команд используйте следующие топики:
* `/home/cabinet/ferrum/ac/cmd/power` — `ON`, `OFF` или `TOGGLE`
* `/home/cabinet/ferrum/ac/cmd/temp` — целевая температура (число от `16` до `32`)
* `/home/cabinet/ferrum/ac/cmd/mode` — режим работы: `cool`, `heat`, `dry`, `fan`, `auto`
* `/home/cabinet/ferrum/ac/cmd/fan` — скорость вентилятора: `low`, `medium`, `high`, `auto`
* `/home/cabinet/ferrum/ac/cmd/display` — управление подсветкой дисплея: `ON`, `OFF`
* `/home/cabinet/ferrum/ac/get` — пустой запрос для принудительного обновления JSON-статуса в топике `/status`

## 💻 Зависимости и библиотеки
Для сборки проекта требуются следующие библиотеки:
* `PubSubClient` — для работы с MQTT.
* `ArduinoJson` — для парсинга и генерации JSON-состояний.
* Кастомная обертка `AcLogic.h` / `mybase_esp32_OTA_webserial.h` на базе инфракрасного стека кондиционеров (протокол COOLIX).
